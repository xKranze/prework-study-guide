HTML

  • The H1, H2, H3, H4, H5, and H6 elements represents the level of text sizes of the heading.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The p element represents a paragraph or block of text.
  • The ul, ol and li elements represents unordered lists, ordred lists and list items. In essence these HTML elements represents bulleted or numbered lines.

Git

  • Enter your Git notes here
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • In the command line, pull down the latest main branch by entering git pull origin main.
  • git fetch really only downloads new data from a remote repository - but it doesn't integrate any of this new data into your working files. Fetch is great for getting a fresh view on all the things that happened in a remote repository. Due to it's "harmless" nature, you can rest assured: fetch will never manipulate, destroy, or screw up anything. This means you can never fetch often enough.
  • git pull, in contrast, is used with a different goal in mind: to update your current HEAD branch with the latest changes from the remote server. This means that pull not only downloads new data; it also directly integrates it into your current working copy files.
